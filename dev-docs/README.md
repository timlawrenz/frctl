# Development Documentation

This directory contains documentation relevant to the development process of frctl itself.

## Development Session Documents

These files track progress across development sessions:

- **SESSION_BRIEF.md** - Current session state, progress tracking, next steps
- **ARCHIVE_SUMMARY.md** - Summary of completed Phase 1 work
- **CONFIGURATION_COMPLETE.md** - Configuration system completion summary
- **DIGEST_COMPLETE.md** - Digest protocol completion summary
- **PERSISTENCE_COMPLETE.md** - Persistence system completion summary
- **PROMPT_TEMPLATES_COMPLETE.md** - Template system completion summary
- **CLI_COMPLETE.md** - CLI implementation completion summary

## Development Process Documents

- **COMMIT_NOTES.md** - Git commit message templates and conventions
- **OPENSPEC_REVIEW.md** - OpenSpec proposal review notes

## Test Scripts

Development test scripts (will be removed before production):

- `test_gemini_direct.py` - Direct Gemini API testing
- `test_gemini_planning.py` - Planning engine testing with Gemini
- `test_gemini_simple.py` - Simple Gemini integration test
- `test_persistence_demo.py` - Persistence system demo
- `list_gemini_models.py` - List available Gemini models

## Working with Development Docs

These documents are **not** part of the user-facing documentation. They are:

1. **Session Continuity** - Help maintain context between development sessions
2. **Progress Tracking** - Track completion of OpenSpec tasks
3. **Development Testing** - Quick scripts for testing during development

For user-facing documentation, see the main `/docs` directory.
